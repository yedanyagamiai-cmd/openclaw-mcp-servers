PART 4: NAMED PROTOCOL PATTERNS (complete list)
================================================================================

From proactive-agent (#2):
  WAL Protocol (Write-Ahead Log): Write SESSION-STATE.md BEFORE responding
    Trigger: Corrections, proper nouns, preferences, decisions, specific values
    Rule: Write → Then Respond (urge to respond is the enemy)

  Working Buffer Protocol: Capture every exchange at >60% context
    Trigger: session_status shows 60%+ context used
    Format: timestamp + Human message + Agent summary per exchange

  Compaction Recovery: Recover after context truncation
    Auto-triggers: <summary> tag, "truncated", "where were we?", "continue"
    Steps: Read buffer → Read SESSION-STATE → daily notes → search → extract

  VBR Protocol (Verify Before Reporting): Test mechanism, not text
    Rule: Text change ≠ behavior change. Verify by observing, not by reading config.
    Anti-pattern: Changing prompt text while keeping systemEvent/isolated unchanged

  ADL Protocol (Anti-Drift Limits): Forbidden evolution rules
    ❌ No complexity to "look smart"
    ❌ No unverifiable changes
    ❌ No vague justifications ("intuition", "feeling")
    ❌ No stability sacrifice for novelty
    Priority: Stability > Explainability > Reusability > Scalability > Novelty

  VFM Protocol (Value-First Modification): Score before implementing
    Dimensions: High Frequency (3x), Failure Reduction (3x), User Burden (2x), Self Cost (2x)
    Threshold: Weighted score < 50 = skip it
    Golden Rule: "Does this let future-me solve more problems with less cost?"

From self-improving-agent (#1):
  Simplify & Harden Feed: Ingest patterns from simplify-and-harden skill
    Uses Pattern-Key for deduplication, tracks Recurrence-Count
    Promote when: Count >= 3 AND seen in 2+ tasks AND within 30 days

  Promotion System: Learning -> permanent workspace files
    Targets: CLAUDE.md (facts) | AGENTS.md (workflows) | SOUL.md (behavior) | TOOLS.md (tools)
