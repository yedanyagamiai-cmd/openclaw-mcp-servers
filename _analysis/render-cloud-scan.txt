SECTION 6 — NOTABLE OBSERVATIONS / RISKS
--------------------------------------------------------------------------------

1. RATE LIMIT NOT PERSISTENT (Intel Pro)
   lib/auth.ts uses in-memory Map for rate limiting. A Render service restart
   or new deployment resets all rate limit counters. Free users get unlimited
   requests across restarts.

2. DEFAULT TOKENS HARDCODED IN SOURCE
   Bunshin: AUTH_TOKEN default 'openclaw-bunshin-2026' in process.env fallback
   Browser: TOKEN = "openclaw-browser-2026" hardcoded at line 3 of index.js
   These are visible in the source repo. If the repo is public, tokens are
   exposed.

3. BROWSER WORKER AUTH CHECK BYPASS ON /
   browser-stealth/src/index.js: auth() is only called AFTER the path check.
   The / route (service info) is publicly accessible without token — confirmed
   by the code flow (auth check at line 144, after path=/ check at line 189).
   Actually re-checking: auth(request) is called at line 144 before any path
   routing, so / IS protected. Correction: the old browser-worker.js has
   "if (path !== '/' && token !== env.AUTH_TOKEN)" — that one allows unauthenticated
   access to /. The v2.0 stealth version correctly gates all paths (returns 401
   before path routing if !auth(request)).

4. DEEPSEEK API KEY IN BUNSHIN
   Bunshin uses api.deepseek.com (native DeepSeek endpoint, not DeepInfra).
   Intel Pro uses DeepInfra. These are different accounts/keys.

5. STRIPE IS PAUSED
   MEMORY.md confirms: "Payment CTA: mailto: (Stripe 暫停中)" — Stripe checkout
   is currently non-functional. Only PayPal and GitHub OAuth flows are active.

6. NO RENDER.YAML FOUND
   Neither service directory contains a render.yaml. Deployments are configured
   manually through the Render dashboard.

7. INTEL PRO TRENDING DATA IS STATIC
   The /api/v1/trending endpoint returns hardcoded data from getTrendingData()
   in lib/r1.ts. It is NOT live-scraped from Smithery/npm/GitHub. Values like
   "total_servers: 11000" are static constants.

8. BUNSHIN DATABASE IS RENDER POSTGRESQL FREE TIER
   MEMORY.md: "PostgreSQL: openclaw-db (Free, expires 2026-04-05)"
   The free PostgreSQL tier on Render expires. If not upgraded, the service
   loses persistent storage after 2026-04-05.

9. PRODUCT STORE D1 DATABASE
   D1 database "gumroad-sales" (id: 1353b53e-000c-4646-ac24-d15cd3fd29ad)
   Orders table presumably stores purchase records.
   GET /api/orders is the verification endpoint per MEMORY.md.
